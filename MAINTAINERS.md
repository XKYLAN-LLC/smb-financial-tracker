# Maintainers

This project is maintained by XKYLAN LLC as an open-source, local-first financial tracking project for small business owners and self-employed people.

| Maintainer | Role |

Maintainers are responsible for keeping the project useful, privacy-safe, and grounded in conservative financial workflows. The project should help users organize records and prepare reviewable exports. It should not present tax, legal, benefits, or insurance determinations as authoritative advice.

## Maintainer Expectations

- Keep public examples synthetic.
- Review pull requests before merging.
- Avoid committing private financial records, tax documents, application IDs, account numbers, SSNs, addresses, phone numbers, emails, screenshots, PDFs, CSV exports, or private seed data.
- Preserve provenance in data-model changes.
- Prefer small, focused pull requests.
- Keep agent instructions clear enough for Codex, Claude Code, and similar tools to follow safely.

## Review Policy

All code, data-model, import, export, agent-skill, and documentation changes should go through pull requests and receive at least one maintainer review before merging to `main`.
